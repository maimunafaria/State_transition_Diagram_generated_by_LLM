Weather Station System — Polished Requirement Specification

Overview
a state diagram for the weather station system that shows how it responds to requests for various services. You can read this diagram as follows: 1. The unlabeled arrow with the black blob indicates that the Shutdown state is the initial state.

Operational Context
Primary actors are not explicitly stated in the source text.

Functional Requirements
1. The system shall respond to a restart() message.
2. The system shall respond to a reconfigure() message when in the Shutdown state.
3. The system shall respond to a powerSave() message when in the Shutdown state.
4. The system shall move to the Summarizing state upon receiving a reportWeather() message.
5. The system shall transmit information to the remote system when in the Transmitting state.
6. The system shall move to the Testing state upon receiving a reportStatus() message.
7. The system shall move to the Transmitting state following the Testing state.
8. The system shall collect data from instruments when in the Collecting state.
9. The system shall move to a controlled state upon receiving a remoteControl() message.
