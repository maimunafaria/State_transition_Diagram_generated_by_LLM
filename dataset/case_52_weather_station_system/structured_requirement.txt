Functional Requirements
1. The system shall support a state diagram for the weather station system that shows how it responds to requests for various services.
2. The system shall support A restart() message causes a transition to normal operation.
3. The system shall ensure that if a shutdown() message is received, the object returns to the shutdown state.
4. The system shall ensure that if a reportWeather() message is received, the system moves to the Summarizing state.
5. The system shall ensure that when the summary is complete, the system moves to a Transmitting state where the information is transmitted to the remote system.
6. The system shall support it then returns to the Running state.
7. The system shall ensure that if a reportStatus() message is received, the system moves to the Testing state, then the Transmitting state, before returning to the Running state.
8. The system shall ensure that if a signal from the clock is received, the system moves to the Collecting state, where it collects data from the instruments.
9. The system shall ensure that each instrument is instructed in turn to collect its data from the associated sensors.
10. The system shall ensure that if a remoteControl() message is received, the system moves to a controlled state in which it responds to a different set of messages from the remote control room.
