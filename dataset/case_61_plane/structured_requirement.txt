Plane — Polished Requirement Specification

Overview
How does a plane object in the state ready for use (1) react to the event «M» assign plane? In order to answer this question, we have to check first if the event «M» assign plane even exists in the state ready for use (1). The event is allowed if a transition (an arrow) to another state exists that is labeled with the event name, or if an internal transition exists (an entry in the lower part of the state symbol).

Operational Context
Primary actors include user.

Functional Requirements
1. The system shall allow a plane object in the state ready for use (1) to accept the event «M» assign plane without changing its state.
2. The system shall allow a plane object in the state scheduled for maintenance (2) to accept the event «M» complete flight and transition to the state in maintenance (5).
3. The system shall prevent a plane object in the state scheduled for maintenance (2) from accepting the event «M» assign flight.
4. The system shall inform the user if assigning a plane fails.
5. The system shall allow a plane object to transition from the state in maintenance (2) to the state ready for use (1) only through the event «M» plane available.
6. The system shall allow a plane object in the state in maintenance (5) to transition out of that state only through the events «M» plane available or «M» plane withdrawn.
7. The system shall only allow the following events to affect a plane object: «M» plane ordered, «M» plane delivered, «M» plane available, «M» plane not available, «M» assign plane, «M» complete flight, «M» scheduling plane for maintenance, and «M» plane withdrawn.
