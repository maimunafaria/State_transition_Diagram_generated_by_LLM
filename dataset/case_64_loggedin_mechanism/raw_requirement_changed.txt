When a user tries to log into the system, they are first asked to enter their username. If they decide to stop at any point, the process is cancelled and they exit the login.

If the username is accepted, the system then asks for the password. Once both the username and password are entered, the system checks whether the details are correct.

If everything is correct, the user is successfully logged in. However, if either the username or password is incorrect, the system lets the user try again.

Each time the user enters incorrect information, the number of attempts increases. The user is allowed only a limited number of tries. If the user keeps entering wrong details and reaches the maximum number of attempts, the system blocks further login attempts and records the failure.

At any point during the process, if the system cannot recognize the user, the login attempt does not proceed.

This approach ensures that users can log in securely while preventing repeated incorrect attempts.
