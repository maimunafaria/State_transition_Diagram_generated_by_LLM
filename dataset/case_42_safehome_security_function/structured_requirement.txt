Safehome Security Function — Polished Requirement Specification

Overview
Each sensor is assigned a number and type, a master password is programmed for arming and disarming the system, and telephone number(s) are input for dialing when a sensor event occurs. When a sensor event is recognized, the software invokes an audible alarm attached to the system. After a delay time that is specified by the homeowner during system configuration activities, the software dials a telephone number of a monitoring service, provides information about the location, reporting the nature of the event that has been detected.

Operational Context
Primary actors are not explicitly stated in the source text.

Functional Requirements
1. The system shall allow configuration of the security system by the homeowner during installation.
2. The system shall monitor all sensors connected to the security system.
3. The system shall interact with the homeowner through the Internet, a PC, or a control panel.
4. The system shall assign a unique number and type to each sensor during system configuration.
5. The system shall allow programming of a master password for arming and disarming the system.
6. The system shall allow input of telephone numbers for dialing when a sensor event occurs.
7. The system shall invoke an audible alarm attached to the system when a sensor event is recognized.
8. The system shall dial a telephone number of a monitoring service after a delay time specified by the homeowner during system configuration activities.
9. The system shall redial a telephone number every 20 seconds until a connection is obtained.
10. The system shall display prompting messages and system status information on the control panel, PC, or browser window.
