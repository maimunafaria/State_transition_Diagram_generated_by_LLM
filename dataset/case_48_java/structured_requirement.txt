Java — Polished Requirement Specification

Overview
consider the state diagram for a part of a Java compiler. The input to the compiler is a text file, which can be thought of as a long string of characters. The compiler reads characters one at a time and from them determines the structure of the program.

Operational Context
Primary actors are not explicitly stated in the source text.

Functional Requirements
1. The system shall ignore white-space characters (e.g., space, tab, newline, and return characters) from input text.
2. The system shall enable the WhiteSpaceAndCommentEliminator object to advance over white-space characters.
3. The system shall enable the WhiteSpaceAndCommentEliminator object to determine whether the next character is part of a comment by checking for ‘/’ followed by another ‘/’ or an ‘*’.
4. The system shall enable the WhiteSpaceAndCommentEliminator object to keep track of whether the current character is white space.
5. The system shall enable the WhiteSpaceAndCommentEliminator object to change state when a ‘/’ is encountered.
