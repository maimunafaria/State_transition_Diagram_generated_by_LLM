E Visa Processing And Follow Up System — Polished Requirement Specification

Overview
A person can start by creating an account or by logging in. If they create an account first, they can then log in. After logging in, they can apply for a visa, view the status, check the status, cancel the application, modify their details, view their details, or view the purpose of the application.

Operational Context
Primary actors are not explicitly stated in the source text.

Functional Requirements
1. The system shall allow a user to create an account.
2. The system shall allow a user to log in after they have created an account.
3. The system shall allow a user to apply for a visa after they have logged in, make the payment and then the application is accepted .
4. The system shall allow a user to view the status after they have applied for a visa.
5. The system shall allow a user to check the status after they have viewed it.
6. The system shall allow a user to cancel an application after they have logged in.
7. The system shall allow a user to modify their details after they have logged in.
8. The system shall allow a user to view their details after they have logged in.
9. The system shall allow a user to view the purpose of the application after they have logged in, Based on the document check, the visa is either approved or rejected, and then the status is updated..
10. The system shall allow a user to give feedback after they have checked the status.
11. The system shall allow a user to log out after they have modified their details, viewed their details, given feedback, or completed a status update.
