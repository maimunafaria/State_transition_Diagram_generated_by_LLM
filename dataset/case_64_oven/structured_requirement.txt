Oven — Polished Requirement Specification

Overview
The statechart shows the different states for cooking food. The initial state is Door Shut. Consider a scenario that starts when the user opens the door.

Operational Context
Primary actors include user.

Functional Requirements
1. The system shall support as a result, the statechart transitions into the Door Open state.
2. The system shall support the user then places an item in the oven, causing the statechart to transition into the Door Open with Item state.
3. The system shall support when the user closes the door, the statechart then transitions into the Door Shut with Item state.
4. The system shall ensure that after the user inputs the cooking time, the Ready to Cook state is entered.
5. The system shall support next the user presses the Start button, which causes the statechart to transition into the Cooking state.
6. The system shall support when the timer expires, the statechart leaves the Cooking state and reenters the Door Shut with Item state.
7. The system shall support if instead the door were opened during cooking, the statechart would enter the Door Open with Item state.
