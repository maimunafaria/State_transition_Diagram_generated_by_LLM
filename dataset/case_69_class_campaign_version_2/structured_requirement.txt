Class Campaign Version 2 — Polished Requirement Specification

Overview
The next step is to prepare a draft state machine for the class. This would need to be expanded in order to reflect any events that have not been identified from the interaction diagrams, and also to include any exceptions. Complex nested states can be refined at this stage.

Operational Context
Primary actors are not explicitly stated in the source text.

Functional Requirements
1. The system shall expand the state machine to include events not identified from interaction diagrams.
2. The system shall refine complex nested states at a later stage.
3. The system shall include an Active state that encompasses Advert Preparation, Scheduling and Running Adverts states.
4. The system shall compare the state machine with use cases to ensure that constraints on class behavior satisfy requirements.
5. The system shall allow a campaign to be cancelled while it is in the Commissioned or Active state.
6. The system shall calculate cancellation costs for billing to the client when a campaign is cancelled.
7. The system shall cancel advertisement schedules when a campaign is active and it is being cancelled.
8. The system shall perform exit activities for concurrent substates when a transition like campaignCompleted fires.
