Functional Requirements
1. The system shall ensure that the next step is to prepare a draft state machine for the class.
2. The system shall support 11.18 results in the addition of the Active state to encompass the states Advert Preparation, Scheduling and Running Adverts (shown in the revised state machine in Fig.
3. The system shall ensure that the state machine is then compared to use cases in order to check that the constraints on class behaviour shown in the state machine satisfy the requirements documented in the use case.
4. The system shall ensure that these might be identified in a final sweep-up to check that the state machine is complete, and could then be added as concurrent states within the Active state.
5. The system shall ensure that let us suppose that further investigation of behaviour that can affect a campaign reveals that in some circumstances a campaign can be cancelled.
6. The system shall ensure that this is not permitted after a campaign has been completed but a campaign can be cancelled while it is in the Commissioned state or in the Active state.
7. The system shall ensure that in either case cancellation costs are calculated for billing to the client.
8. The system shall ensure that if the campaign is active then advertisement schedules are also cancelled.
9. The system shall ensure that in this version the transition campaignCompleted is shown explicitly from the nested concurrent substate Running Adverts to the state Completed.
10. The system shall ensure that when a transition like this fires, any exit activities for the other concurrent substates that are occupied are performed.
