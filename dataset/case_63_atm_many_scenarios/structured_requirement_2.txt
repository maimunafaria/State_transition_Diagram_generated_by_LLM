Atm Many Scenarios Scenario 2 — Polished Requirement Specification

Overview
A person walks up to the ATM, which is waiting for someone to use it. They insert their card, and the machine asks for a PIN. The user enters a PIN, and the system checks whether it is correct.

Operational Context
Primary actors include user.

Functional Requirements
1. The system shall prompt the user to enter a PIN after card insertion.
2. The system shall verify the entered PIN against stored information.
3. The system shall inform the user if the PIN is incorrect and request another attempt.
4. The system shall limit the number of failed PIN attempts before retaining the card for security reasons.
5. The system shall retain the card if it is invalid (such as expired or reported stolen) for security reasons.
6. The system shall display available options to the user upon successful PIN entry.
