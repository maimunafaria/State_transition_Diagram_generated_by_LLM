Atm Many Scenarios Scenario 5 — Polished Requirement Specification

Overview
A user reaches the ATM and proceeds to choose the withdrawal option. The machine begins processing the request and checks with the system to see if the transaction can be approved. If everything is fine, the ATM dispenses the cash.

Operational Context
Primary actors include user.

Functional Requirements
1. The system shall check with the ATM to approve the withdrawal request.
2. The system shall ensure that if the withdrawal is approved, the system shall instruct the ATM to dispense cash.
3. The system shall support after dispensing cash, the system shall instruct the ATM to print a receipt.
4. The system shall support after completing the transaction, the system shall instruct the ATM to return the card.
5. The system shall support after completing the session, the system shall reset the ATM and make it ready for the next user.
6. The system shall ensure that if the withdrawal request is rejected during processing, the system shall instruct the ATM to cancel the operation and return the card.
7. The system shall ensure that if the machine does not have enough cash after approval, the system shall stop the operation and indicate that the system is unavailable.
