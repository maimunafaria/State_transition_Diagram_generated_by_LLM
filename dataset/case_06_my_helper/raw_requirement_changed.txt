A user opens the app to use it.

If the user is new or has forgotten their password, they create a new password. Once the password is set correctly, they can continue using the app.

If the user cannot use their own phone, they send a message from another mobile. After the message is received, it is read to understand what the user needs.

The user’s password is then checked. If the password is wrong, the user is informed through a message, and nothing further happens.

If the password is correct, the user’s request is checked. If the request is valid, it is completed, and the required details are sent back to the user through a message.

If the request is not valid, the user is informed through a message that the request cannot be accepted.
