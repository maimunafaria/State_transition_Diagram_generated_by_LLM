Controlpanel Class — Polished Requirement Specification

Overview
As part of the activation procedure, the ControlPanel object must determine whether any sensors are open. A responsibility named determine-sensor-status() is defined. If sensors are open, ControlPanel must set a status attribute to “not ready.” Sensor information can be acquired from each Sensor object.

Operational Context
Primary actors are not explicitly stated in the source text.

Functional Requirements
1. The system shall ensure that as part of the activation procedure, the ControlPanel object must determine whether any sensors are open.
2. The system shall ensure that if sensors are open, ControlPanel must set a status attribute to “not ready.” Sensor information can be acquired from each Sensor object.
3. The system shall ensure that when the review leader comes to “control panel,” in the use case narrative, the token is passed to the person holding the ControlPanel index card.
4. The system shall ensure that the phrase “implies that a sensor is open” requires that the index card contains a responsibility that will validate this implication (the responsibility determinesensor-status() accomplishes this).
5. The system shall ensure that next to the responsibility on the index card is the collaborator Sensor.
6. The system shall ensure that the token is then passed to the Sensor object.
7. The system shall ensure that when the token is passed, the holder of the Sensor card is asked to describe the responsibilities noted on the card.
8. The system shall ensure that if the responsibilities and collaborations noted on the index cards cannot accommodate the use case, modifications are made to the cards.
