Placement Automation System — Polished Requirement Specification

Overview
Placement provides the opportunity for the students to gain Specific Subject skills as well as Real work employability skills. Placement Offices play a crucial role in providing students the necessary training and various opportunities where the students can effectively evidence their various skills and abilities on their resume. Hence, the Automation of Placement office provides a more smooth placement process for both the companies and the students.

Operational Context
Primary actors include admin, user, student, companies, company.

Functional Requirements
1. The system shall allow Admin to add a new company.
2. The system shall allow Admin to delete or update a company.
3. The system shall keep track of students interested in placements for Admin.
4. The system shall allow Admin to add a new training for students.
5. The system shall allow Admin to update or delete a training.
6. The system shall allow User to register for a particular job.
7. The system shall allow User to register for a particular company.
8. The system shall allow User to undertake a training.
9. The system shall allow User to submit a resume.
10. The system shall allow User to update their resume.
11. The system shall allow Company to register in the placement office.
12. The system shall allow Company to view a student's resume.
13. The system shall allow Company to recruit a student with a good resume and relevant skills.
14. The system shall allow Company to recruit a student trained in a specific area.
