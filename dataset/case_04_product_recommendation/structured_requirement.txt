Product Recommendation — Polished Requirement Specification

Overview
This case study focuses on using web scrapping for product comparison across websites to reduce the amount of time and effort involved in finding and comparing the products. 1. Register Description: Allows the user to register/login to use the system.

Operational Context
Primary actors include user.

Functional Requirements
1. The system shall allow users to register and login.
2. The system shall display search results for products.
3. The system shall show more relevant results when searching within a specified website.
4. The system shall display more relevant results when filtering products by user-selected criteria.
5. The system shall enable users to purchase products.
