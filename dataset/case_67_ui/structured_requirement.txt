Ui — Polished Requirement Specification

Overview
The requirement here is that the users must be able to check whether the budget for an advertising campaign has been exceeded or not. This is calculated by summing the cost of all the adverts in a campaign, adding a percentage for overheads and subtracting the result from the planned budget. A negative value indicates that the budget has been overspent.

Operational Context
Primary actors include user.

Functional Requirements
1. The system shall calculate the budget balance by summing advert costs, adding an overheads percentage, and subtracting from the planned budget.
2. The system shall display a list of clients in the Client dropdown when a client is selected.
3. The system shall enable the Campaign dropdown with a list of campaigns after a client has been selected.
4. The system shall enable the Check button when a campaign is selected.
5. The system shall display the result of the budget calculation in the Budget textfield.
6. The system shall perform a budget balance calculation when the Check button is clicked.
7. The system shall clear the Budget textfield when a new client or campaign is selected.
8. The system shall disable the Check button when a new client is selected.
9. The system shall display an alert dialogue with the message ‘Close window? Are you sure?’ and OK/Cancel buttons when the Close button is clicked.
10. The system shall exit the use case when the OK button is clicked in the alert dialogue.
