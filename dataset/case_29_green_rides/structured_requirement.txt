Green Rides — Polished Requirement Specification

Overview
A user starts by logging in and selecting where they want to go. The system shows the journey details, and the user can check them again if needed. From there, the user can turn on driving mode or continue with other options.

Operational Context
Primary actors include user, authority.

Functional Requirements
1. The system shall display journey details upon user login and destination selection.
2. The system shall allow users to activate driving mode or choose other options after viewing journey details.
3. The system shall enable users to select a place to eat, view its menu, and place an order if they are hungry.
4. The system shall inform relevant authorities in the event of an accident.
5. The system shall send notifications during driving or in case of an accident.
6. The system shall allow users to provide feedback upon completion of their journey.
