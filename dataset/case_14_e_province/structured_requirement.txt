E Province — Polished Requirement Specification

Overview
Citizens in every district face the difficulty with the of government certificates like birth/death certificates, income and caste certificates, old age and widow pensions, etc. This can be done with e-Province. This case study demonstrates how make the government services available to the citizens in a convenient, efficient and transparent manner, by enabling it simple by reliable access over the Internet.
The main objective is to bring the public services closer to the citizens.

Operational Context
Primary actors include admin, user, citizens, authority.

Functional Requirements
1. The system shall allow new users to the register to e-Province Website online and verify the account once the user is verified.
2. The system shall allow the authorized users to login to the website using their unique ID.
3. The system shall create the forms for the required certificate that the user has requested and when the users filled the details with the valid prerequisites, the registration for certificate process completes.
4. The system shall validate the details of the form and issues the certificates if it is approved.
5. The system shall ensure that the main objective is to bring the public services closer to the citizens.
6. The system shall ensure that fulfil the vision of providing an easy and convenient service to the citizens through remote access primarily through Common Service Centre (CSCs), e-Province project is created.
7. The system shall ensure that the various modules in the e-Province project are an Admin module that creates, updates, deactivates the users and performs various other tasks.
8. The system shall allow A Login module that allows the users to login with their unique Application number with approved status.
9. The system shall allow A User module that allows the approved users to register for the respective certificates.
10. The system shall support A User Profile module that creates the pre-requisite information of the user that needs to be stored in the database.
11. The system shall ensure that the Certificate module contains the required forms that are to be filled with the information of the users .
12. The system shall allow registration Process Description: This process allows new users to the register to e-Province Website online and verify the account once the user is verified.

Behavioral Conditions
Preconditions:
- Users should possess the prerequisites (e.g. Voter ID and Aadhaar card) for registering.
- User should possess the verified unique ID.
- User should possess the details that are requested by the form to be filled (e.g. Aadhaar card, Voter ID).
- Document is uploaded by the user to update the profile.
Postconditions:
- The details are uploaded to the database and the user is provided with a unique ID.
- Logged in Successfully.
- Form is filled with the required details and he requests the certificate from the issuer. Certificate Issuing Process.
- The certificate is issued to the user. User Profile Viewing Process.
- The complete user profile is displayed in the portal.
Triggers:
- Registering to the e-Province and creating the unique ID in which the data are stored in the e-Province database.
- Login with the verified unique ID.
- If the unique ID is not present in the database, it shows an error message.
- The user can provide the details of the required certificate that is requested by him, and the data are stored in e-Province database for further process.
- If the details are not provided correctly it displays an error message.
- The authority validates the details provided by the user and issues the required certificate.
- User can view his profile whenever needed and he is intimated by the portal if he received the certificate.
