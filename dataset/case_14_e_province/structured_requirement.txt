E Province — Polished Requirement Specification

Overview
Citizens in every district face the difficulty with the of government certificates like birth/death certificates, income and caste certificates, old age and widow pensions, etc. This can be done with e-Province. This case study demonstrates how make the government services available to the citizens in a convenient, efficient and transparent manner, by enabling it simple by reliable access over the Internet.

Operational Context
Primary actors include admin, user, citizens, authority.

Functional Requirements
1. The system shall allow new users to register online and verify their account.
2. The system shall create a unique ID for each registered user and store data in the e-Province database.
3. The system shall allow authorized users to login using their verified unique ID.
4. The system shall display an error message if the unique ID is not present in the database.
5. The system shall log users in successfully when they login with their verified unique ID.
6. The system shall create forms for required certificates and store data in the e-Province database.
7. The system shall display an error message if the details are not provided correctly.
8. The system shall allow users to request certificates and store data in the e-Province database.
9. The system shall validate user details and issue certificates if approved.
10. The system shall issue certificates to users when validated.
11. The system shall allow users to view their profile and update it when a certificate is approved.
12. The system shall display complete user profiles in the portal.
