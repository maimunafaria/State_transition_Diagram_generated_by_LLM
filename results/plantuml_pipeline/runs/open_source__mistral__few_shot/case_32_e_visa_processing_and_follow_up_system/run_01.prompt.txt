Act as a software requirements analyst and UML modeling expert.



Your task is to generate a UML state transition diagram in PlantUML format from the given natural language requirement.



Follow the structure demonstrated in the examples below.



--- Examples ---

Example 1 Requirement:
Cd Player — Polished Requirement Specification

Functional Requirements
1. The system shall start in a stopped state.
2. The system shall begin playing music when a disc is inserted and the user presses play.
3. The system shall allow the user to pause the music at any time while it is playing.
4. The system shall stop the music temporarily when paused but allow resuming from the same point.
5. The system shall allow the user to stop the player at any time, regardless of whether the music is playing or paused.
6. The system shall return to a stopped state when the player is stopped.
7. The system shall allow switching to another disc and continuing playback if multiple discs are available.
8. The system shall remain in a stopped state when no more discs are left.

Example 1 PlantUML:
@startuml
title CD Player State Machine

[*] --> Stopped

state Playing
Playing : do / read disc

state Stopped
state Paused

Stopped --> Playing : press play [disc in tray]
Playing --> Stopped : press stop
Playing --> Stopped : [no more discs]

Playing --> Paused : press pause
Paused --> Playing : press play
Paused --> Stopped : press stop

Playing --> Playing : [another disc] / change disc

@enduml

3. The system shall check the entered username against its records.
4. The system shall prompt for a password if the username is valid.
5. The system shall verify both the username and password to allow login.
6. The system shall allow the user to try again if the credentials are incorrect.
7. The system shall increase the number of failed attempts with each incorrect entry.
8. The system shall limit the number of login attempts for a user.
9. The system shall block further login attempts if the maximum number of attempts is reached.
10. The system shall prevent login if it cannot recognize the user during any step.

Example 2 PlantUML:
@startuml
title LoggingOn State Machine

state LoggingOn {
  [*] --> AcceptingUserName

  state AcceptingUserName
  state AcceptingPassword
  state VerifyingDetails

  state BadDetails {
    state BadUsername
    state BadPassword
  }

  BadDetails : entry / ++tries

  AcceptingUserName --> AcceptingPassword : ok

  AcceptingUserName --> Canceling : cancel

  AcceptingPassword --> VerifyingDetails : ok
  AcceptingPassword --> Canceling : cancel

  VerifyingDetails --> BadUsername : badUserName
  VerifyingDetails --> BadPassword : badPassword
  VerifyingDetails --> Authenticated : detailsVerified

  BadDetails --> AcceptingUserName : [tries < 4]
  BadDetails --> Fail : [tries = 4]
}

LoggingOn : entry / tries := 0

state Canceling
state Authenticated
state Fail
Fail : entry / logFailure
userRecognized --> AcceptingPassword
@enduml

9. The system shall upload a final project once work is completed.
10. The system shall perform a quality check on the final project.
11. The system shall restart the process if a client is unsatisfied with the final project.
12. The system shall make payment if the client is satisfied with the final project.
13. The system shall deliver the project after payment.
14. The system shall allow a client to provide a rating for the completed project.

title UML State Machine Diagram (Upload or Accept Request)

state "Upload or Accept request" as UAR {

    UAR : exit / After finishing rating

    ChoiceUser --> SearchCategory : [user = client]

    UpdateDB --> [*]


    state "Search project category" as SearchCategory
    state ChoiceBPO <<choice>>

    SearchCategory --> ChoiceBPO

    ChoiceBPO --> EndDecline : [BPO org declines]
    ChoiceBPO --> ChoiceClient : [BPO org accepts]

    ChoiceClient --> EndReject : [client rejects]
    ChoiceClient --> UpdateStatus : [client accepts]

    state EndDecline <<choice>>
    EndDecline --> [*]

    state EndReject <<choice>>
    EndReject --> [*]

    state "Update status of project" as UpdateStatus

    ChoiceTracking --> DisplayProgress : [client wishes to track]
    ChoiceTracking --> UpdateProgress : [user = BPO org]

    state "Display progress" as DisplayProgress
    DisplayProgress --> [*]

@enduml



--- Process ---

1. Identify all system states.

2. Identify events/conditions triggering transitions.

3. Define transitions clearly between states.

4. Ensure logical consistency (no unreachable states).

5. Identify one initial state and at least one final state.



--- Output Rules ---

- Generate ONLY valid PlantUML code.

- Include initial and final states.

- Use proper UML state diagram syntax.

- Do not include explanations or extra text.
