Act as a software requirements analyst and UML modeling expert.



Your task is to generate a UML state transition diagram in PlantUML format from the given natural language requirement.



Follow the structure demonstrated in the examples below.



--- Examples ---

Example 1 Requirement:
Guide Dropshipping Website — Polished Requirement Specification

Functional Requirements
1. The system shall allow the user to choose whether they are a seller or a customer.
2. The system shall enable the seller to add or remove products from their list and finalize their offerings.
3. The system shall allow the customer to view available products and select ones of interest.
4. The system shall facilitate communication between customers and sellers for deal discussions.
5. The system shall determine whether a purchase is completed based on the discussion or if it does not proceed.
6. The system shall allow the customer to make payment after a purchase is completed and provide feedback later.
7. The system shall end the interaction if no deal occurs.

@startuml

StartChoice --> CustomerLogin

SellerLogin : entry / seller should be registered

SellerLogin : exit / login Successful

state "Add/Remove Products" as ManageProducts
ManageProducts : entry / seller should be LoggedIn
ManageProducts : do / the seller should be able to add and remove products from the lists
ManageProducts : exit / fthe seller finalise his products list

SellerLogin --> ManageProducts


state "Customer Login" as CustomerLogin
CustomerLogin : entry / customer should be registered
CustomerLogin : do / user provide Login Details
CustomerLogin : exit / login Successful

state "View Products" as ViewProducts
ViewProducts : entry / customer should be LoggedIn
ViewProducts : do / the customer view Product List
ViewProducts : exit / customer decides to buy a product

state "Purchase Products" as PurchaseProducts
PurchaseProducts : entry / customer select a Product
PurchaseProducts : do / customer contacts Seller via messaging
PurchaseProducts : exit / customer should be able to message the seller
CustomerLogin --> ViewProducts
ViewProducts --> PurchaseProducts


state MergePoint <<choice>>

ManageProducts --> MergePoint

--- Process ---

1. Identify all system states.

2. Identify events/conditions triggering transitions.

3. Define transitions clearly between states.

4. Ensure logical consistency (no unreachable states).

5. Identify one initial state and at least one final state.



--- Output Rules ---

- Generate ONLY valid PlantUML code.

- Include initial and final states.

- Use proper UML state diagram syntax.

- Do not include explanations or extra text.
