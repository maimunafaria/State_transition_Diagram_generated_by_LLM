In an application, users should be able to easily check information through a simple and guided interface. To support this, the system provides a feature that allows users to open a budget-checking window from the main screen.

When the user selects the option from the main window, a new window is opened where the checking process takes place. At first, no client is selected, so the system waits for the user to choose one. The user cannot proceed until a client is selected.

After selecting a client, the system then requires the user to select a campaign related to that client. If no campaign is selected, the system continues to wait and does not perform any action.

Once both the client and campaign are selected, the system allows the user to check the information. Initially, nothing is shown in the result area. When the user presses the check button, the system displays the corresponding result. The user can press the check button again if they want to refresh or recheck the information.

At any time, the user may choose to close the window. When the close button is pressed, the system shows a confirmation message asking whether the user wants to continue or cancel the action. If the user confirms, the system returns to the main screen. If the user cancels, the window remains open so the user can continue working.

This design ensures that the user is guided step by step, preventing mistakes and making the interaction simple and user-friendly.
