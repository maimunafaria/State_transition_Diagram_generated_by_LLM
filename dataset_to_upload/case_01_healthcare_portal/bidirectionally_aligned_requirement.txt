A user accesses the healthcare platform. If the user is new, the user first completes registration. If the registration is unsuccessful, access ends there. If the registration is successful, or if the user is already registered, the user signs in to the system.

After signing in, the user can access the available facilities. From there, the user may choose to read news and expert content, explore health-related information, or continue with finance-related services.

When the user chooses health-related information, the user can either check symptoms to get possible disease-related results or use the fitness checking option to calculate fitness-related results. After viewing the result, the user may continue using the system or finish the session.

When the user chooses finance-related services, the next steps depend on whether insurance is available. If insurance is already available, the insurance is processed and validated. If insurance is not available, the user goes through the application process and the provided details are validated before insurance is provided.

After that, the user proceeds depending on the type of service used. If the user has consulted a doctor, the user goes through the payment process, where insurance details are checked and payment is completed. If the user has chosen drugs and supplements, the user selects how the payment will be handled. If the selected payment type is cash, the amount and return date are prepared and the user is informed to pay at the time of delivery. If the selected payment type is through insurance, the payment process is completed by checking insurance details.

Once payment is completed, the user may choose whether to provide feedback. If the user chooses to do so, feedback is collected for the service. After that, the user finishes the session and logs out of the system.
