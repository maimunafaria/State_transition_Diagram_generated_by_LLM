Title: State and Event Glossary for Generalized Requirements

Purpose:
Map varied requirement wording into consistent state/event vocabulary.

State synonym hints:
- start, initial, init -> Initial
- waiting, pending, queued -> Pending
- in progress, processing, ongoing -> Processing
- done, completed, success -> Completed
- failed, error, exception -> Failed
- stopped, cancelled, aborted -> Cancelled
- paused, on hold -> Suspended

Event synonym hints:
- submit, send, place request -> submit
- approve, accept, authorize -> approve
- reject, decline, deny -> reject
- cancel, abort, stop -> cancel
- retry, try again, reattempt -> retry
- timeout, no response -> timeout
- login, sign in -> login_request
- logout, sign out -> logout

Required usage guidance:
- Keep one canonical event label per intent in a single diagram.
- Prefer explicit event labels over unlabeled arrows when text evidence exists.

Forbidden usage guidance:
- Avoid mixing tense forms for the same event (approve/approved/approving).
- Avoid vague labels like action or process unless unavoidable.

PlantUML mini example:
@startuml
[*] --> Pending
Pending --> Processing : submit
Processing --> Completed : approve
Processing --> Failed : reject
Failed --> Processing : retry
Completed --> [*]
@enduml
